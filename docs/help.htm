wkbre v0.0.4.0 Help file

Introduction

wkbre is a program in which you can load, edit and save saved games of Warrior Kings (Battles). Additionally, it loads the game set, including the .cpp files, meshes, textures, …, so you can get a 3D view looking exactly as in the game.

But wkbre is currently incomplete. There may be some bugs in it. It may not load and save games properly, it cannot edit everything and the landscape has only a green texture.

The source code is available here: https://github.com/AdrienTD/wkbre

Start

If wkbre doesn’t start, be sure that the Visual C++ 2010 Redistributable Package (x86) and the latest DirectX runtime are installed. If not, you can download them from http://www.microsoft.com/download .

First you need to configure the "settings.txt" file. Ensure that the GAME_DIR value is the directory of the game (with "data.bcp" and the "saved" folder). The other settings are not important.

After setting the GAME_DIR value you can start the executable (wkbre.exe). You will be asked which saved game you want to load. Select a file in the list then click OK or double-click on the file to open it.

The landscape may look incorrect. If it does, press the '0' key repeatedly until the landscape is correct.

Use the arrow, ‘E’ and ‘D’ keys to move the camera.

Click an object to select it. Press the Control key and the arrow keys at the same time to move the selected object. You can also press the Insert key to duplicate the selected object, the Delete key to delete it, and the * or / key to change the size. Look at the menu bar in the program or the Keys section of this document for a list of available commands.

Once you have finished doing some modifications in the level, press F3 and type the name of the new savegame. The name should end with “.sav”, otherwise the game won’t display it in its list.

Settings

In “settings.txt”:

SettingValue
GAME_DIRGame directory (in quotation marks)
FAR_Z_VALUEThe greater the value, the farther you can see.
OCCLUSION_RATEThe greater the value, the more you can see.
MAP_MAX_PART_SIZEMaximum width and height (each in number of tiles) of a single part of the landscape.
HARDWARE_VERTEX_PROCESSING1 for hardware, 0 for software vertex processing
TEXTURE_COMPRESSION1 = texture compression (DXT1) enabled, 0 = disabled
(NOTE: In version 0.0.3, texture compression is always disabled. This may come back in future releases.)
VSYNC1 = vertical synchronization enabled, 2 = high resolution VSYNC enabled, 0 = disabled
FOG1 = fog enabled, 0 = fog disabled
FONTPath to a .font file used to draw text (Don’t forget the quotation marks!)
HARDWARE_CURSOR1 = Use the system cursor, 0 = Draw cursor in game screen buffer
SCREEN_SIZEInitial size of the game screen/window size. Requires two numbers for width and height in pixels.
FULLSCREEN0 = windowed, 1 = fullscreen (NOTE: Fullscreen mode buggy at the moment.)

Known issues

  • The landscape looks incorrect when using WKBattles files.
    Press the '0' key repeatedly until the landscape is correct.
  • Some parts of the manor and the village are black.
    To correct this issue, go to the settings.txt file and change the value of TEXTURE_COMPRESSION to 0.
    •
  • Texture compression is disabled even with TEXTURE_COMPRESSION 0 in settings.txt
    The texture loading in version 0.0.3 has been rewritten, and the new code doesn’t support texture compression for various reasons.
  • I don't see the landscape!
    The landscape mesh has too many primitives and your video card doesn’t support that. In “settings.txt” look for MAP_MAX_PART_SIZE and set lower values.
  • wkbre is frozen / stops drawing.
    Minimize and restore the window.
  • When I load a savegame made with wkbre in the original engine, there are some issues. The AI doesn’t work well, the formations move poorly, …
    This is going to be fixed in a next release.
  • I don’t find my savegame in WKB.
    Look at every tab (skirmish, campaign, multiplayer).

Playing games in wkbre

If you click Help > Enable experimental features, you can enable new features that were not initially available as they are buggy, complex and you can easily make mistakes by pressing an incorrect keyboard button.

The notable experimental feature is the game playing. This is done by interpreting the game set script which can be found in the game data (data.bcp). When you press 'P', the game will be unpaused. You can see your units moving, peasants doing their work, etc...

When you have selected an unit you can also give it an order by right-clicking a target. Press 'A' to execute button commands.

Please note that savegames with AI players might not work because some game set script features related to AI are not supported yet.

If you have opened a lvl file, you can press 'S' to start the level (for example in a tutorial this will show the first message).

See the Keys section of this document for list of available experimental features.

Also please keep in mind that playing games in wkbre is "experimental", this means the feature is not finished and may contain bugs. wkbre cannot interpret the whole game script at the moment. It is possible to get errors like "Action ... is unknown." which means a certain action or other feature of the game script language has not been implemented yet.
